    基本情况

    • 综合模型重构、模型输入及输出优化、多线程并行及硬件特性加速等方法，最终实现了接近500倍的加速， 实现了算法的终端部署；
    • 根据目标位置相关性分析设计了快速高效的乘客多目标跟踪算法及乘客上下车统计算法；
    • 进行算法优化时，对CPU整体计算性能、访存性能、算法热点进行了详细的测试，采用了选择热点，逐步优 化的优化思路；
    • 为实现客流统计的多级流水技术及多线程并发，基于pthread封装了大量多线程开发基础类，包括线程、互 斥量、条件变量、阻塞队列、双缓冲区、线程池等；
    • 数据采集及视频编码优化，采用多级流水的方式进行整体方案的设计，针对图像处理及视频编码使用了硬件 加速；
    • 制定硬件需求方案，完成硬件的软件配置，算法运行环境配置，并将算法移植到目标设备中；

    2015.11-2017.1 国际金融衍生品交易系统V9.0 项目设计及开发

    项目简介

    系统采用经典的金融交易系统的设计模式，统分为多个子系统，是典型的分布式设计模式，包括交易前置 子系统，管理前置子系统， 交易子系统，上手网关子系统， 自动结算子系统， 管理子系统， 外围包括行情 系统， 管理终端，交易终端，API接入，级联子系统等。

    基本情况

    • 前置系统使用了事件驱动状态机模型（EDSM）解决高并发问题，基于epoll实现，出于性能考虑使用了可调 节的回调线程池来保证数据的及时读取及发送，同时采用数据发送缓冲区来提高数据的发送性能；
    • 前置系统使用使用了多进程及多线程的设计模式，进程或线程间使用了双缓冲消息队列作为交互工具，具有 极高的性能，线程间数据独立设计，除数据交互外均无锁；
    • 前置系统实现上采用数据流上下行分离、数据持久化分离、会话流及通知流模式，并采用了序列号的流控模 式；
    • 前置系统设计了高效的业务心跳模式，实测性能极佳；采用客户端维持心跳，每隔10s主动发送一次心跳请 求，前置服务接受到请求后响应，双向检测；前置系统实现了独立的基于定时器的心跳检测模块，消除了心 跳检测带来的延迟；
    • 为实现数据的快速分发，前置服务采用了散列表及备忘录的设计，实现了常数级效率的数据分发，快于基于 Trie树的分发模式；
    • 主导管理API接入设计，独立开发管理API接入模块，并负责管理API文档编写及客户的部分使用疑难问题；
    • 参与交易API的测试，并编写自动可视化测试工具辅助测试；
    • 参与上手网关子系统的设计，并独立完成SGX、DGCX、CQG、IB、ES内外盘子系统接入等网关模块的设计及实 现；
